# MathNotes

## Purpose

This repo converts handwritten math scans into organized Markdown documents for an Obsidian vault.

## Skills

Invoke these skills for specific workflows:

| Skill | When to use |
|-------|-------------|
| `process-scans` | Scans present in `sans_to_parse/` — runs the full transcription pipeline |
| `verify-wikilinks` | Audit or fix `[[wikilinks]]` across the vault |
| `new-math-note` | Create a math note from scratch (no scan source) |
| `add-diagram` | Add a Mermaid concept diagram to an existing note |
| `discover-connections` | Find missing insights and bridging concepts between vault notes |

## Obsidian Vault

The `notes/` directory is an **Obsidian vault**. All output files live there and must be Obsidian-compatible.

- **Wikilinks must use piped format**: `[[kebab-filename|Display Name]]`. Never bare alias `[[Display Name]]` — creates phantom graph nodes.
  - Correct: `[[tannerys-theorem|Tannery's Theorem]]`
  - Wrong: `[[Tannery's Theorem]]`
- **Aliases**: Every note must include `aliases` in frontmatter mapping natural title(s) to the kebab-case file.
- Tags use Obsidian format: `tags: [real-analysis, convergence]`
- Headings hierarchical: `#` > `##` > `###`
- `notes/Odds & Ends/` is non-math content — only touch when explicitly asked.

## Note Format

```markdown
---
aliases: [Natural Title of Note]
tags: [topic-tag, course-tag]
source: sources/YYYY-MM-DD/filename.jpeg
date: YYYY-MM-DD
---

# Title

Body with LaTeX math...

## See Also
- [[kebab-filename|Related Concept]]
```

- Files in `notes/`, named in kebab-case: `uniform-convergence.md`
- Multi-topic scans split into separate files

## LaTeX Conventions

- `$...$` inline, `$$...$$` display (not `\[...\]`)
- `\text{}` for words in math: `$x \text{ is rational}$`
- Standard commands: `\lim`, `\sup`, `\inf`, `\cos`, `\sin`, `\to`, `\infty`, `\epsilon`, `\leq`, `\geq`
- Piecewise: `\begin{cases}...\end{cases}`
